# Звіт про результати тестування

### 1. Unit-тестування
* Повністю покрито модулі `Users` та `Posts` (рівень Services).
* Бізнес-логіка ізольована від зовнішніх залежностей за допомогою моків Prisma Client. 
* Перевірено всі базові операції: створення, пошук за ID (включаючи обробку помилок) та отримання списків.

### 2. Інтеграційне тестування
* Реалізовано автоматичний запуск повного стеку перед виконанням тестів.
* У блоці `beforeAll` ініціалізується інстанс додатка Fastify, встановлюється з'єднання з базою даних PostgreSQL та запускається HTTP-сервер.
* Використання `supertest` для перевірки реальних HTTP-запитів, що підтверджує коректну роботу всього ланцюжка: *Route -> Controller -> Service -> Database*.

### 3. E2E тестування
* Охоплює повний цикл взаємодії користувача з системою.
* Реєстрація нового користувача -> Створення публікації (Post) з прив'язкою до цього користувача -> Перевірка відображення публікації у загальному списку.
* Підтверджено цілісність системи та коректність зв'язків між сутностями на рівні БД.

### 4. Мутаційне тестування
